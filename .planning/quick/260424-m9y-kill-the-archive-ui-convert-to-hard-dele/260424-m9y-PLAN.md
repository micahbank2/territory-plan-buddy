<objective>
Kill the archive UI. Archive is a data-loss trap today: `remove()` already does hard `.delete()` while the UI advertises reversible archival via `restore`/`permanentDelete`/`loadArchivedData` stubs that are no-ops. Users who press "Archive" see a "📦 archived" toast but the record is gone forever with no restore path.

Replace the entire archive concept with explicit hard-delete behind a strong AlertDialog confirmation. This aligns the app with its Core Value: "The app must never silently lose data. Every edit the user makes must either persist to Supabase or visibly fail with a clear error."

Purpose: Eliminate the gap between what the UI promises (recoverable archive) and what the code does (permanent delete). Either commit to true soft-delete (requires schema change — deferred) or remove the false promise. This plan removes the false promise.

Output:
- `useProspects.ts` stripped of archive stubs and `archivedData` state
- `TerritoryPlanner.tsx` stripped of `showArchive`, archive Dialog, archive toolbar button, archive overflow-menu item; row/sheet delete paths gated by AlertDialog
- `ProspectPage.tsx` delete path gated by AlertDialog
- Tests updated (no more `it.todo` entries referencing DATA-06/07/08 soft-delete, archive view test)
- CLAUDE.md gotcha #9 updated to reflect new reality
</objective>
