## Interaction Contracts

### 1. Pending Outreach Badge (D-06)

- Placement: Notification dot or numeric Badge rendered directly on the "Draft Emails" button in the header toolbar.
- Badge renders only when a pending batch exists in localStorage (key: `tp-pending-outreach`).
- Badge displays the count of contacts in the pending batch (e.g., "7").
- Badge uses shadcn `Badge` with `variant="destructive"` for visibility (red dot draws attention without implying danger — acceptable for a personal tool).
- Clicking the Draft Emails button when a badge is present: opens Pending Outreach dialog instead of (or before) the contact picker. Decision: show a split choice — "View Pending (7)" vs "Start New Draft" — as two buttons in the dialog trigger area, or open Pending Outreach dialog directly if a batch exists. Use the latter (simpler): if batch exists, button opens Pending Outreach dialog; a secondary link "Start new draft" appears inside that dialog.

### 2. Pending Outreach Dialog (D-07, D-08)

- Trigger: Draft Emails button when pending batch exists.
- Dialog width: `max-w-lg` (512px). Full-width on mobile.
- Title: "Pending Outreach" (20px semibold).
- Subtitle: "Review who you sent emails to and confirm." (14px muted).
- Content: Grouped list of accounts. Each account shows account name as a section label (12px medium, muted). Under each account, each contact row shows: Checkbox + Contact name + Contact title.
- "Select All" / "Deselect All" toggle link at top of list (12px, muted-foreground).
- Footer: Two buttons — "Mark as Sent" (primary, accent) + "Cancel" (ghost). "Mark as Sent" is disabled when zero contacts checked.
- "Mark All Sent" shortcut: A secondary text link "Mark all as sent" inline near the "Select All" toggle for the common case (D-specific call-out in CONTEXT.md: "Mark All Sent should be prominent").

### 3. Save Behavior (D-09, D-10)

- On "Mark as Sent" click: show a loading state on the button (spinner inside button, button text "Saving...").
- Success: dialog closes, batch cleared from localStorage, toast success: "Logged {N} outreach interactions."
- Error: toast error: "Failed to log some interactions. Check your connection and try again." Dialog stays open.
- Stage bump is silent — no per-row toast for the outreach field update.

### 4. Bulk Mark Contacted — Main Table (D-11, D-12)

- Location: Bulk action bar (already visible when `selected.size > 0` in TerritoryPlanner).
- New button label: "Mark Contacted" with `Mail` icon (lucide-react).
- Button variant: `outline` to distinguish from destructive "Delete" while remaining visible.
- Placement: After "Bulk Edit" button, before "Delete" button in the bulk bar.
- Clicking: Shows a confirmation inline in the bulk bar (no extra dialog): "Log Email + bump stage for {N} accounts? [Confirm] [Cancel]". Confirm proceeds, Cancel dismisses.
- Success toast: "Logged outreach for {N} accounts."
- Error toast: "Failed to log outreach for some accounts. Reload to verify."

### 5. States for Contact Rows in Dialog

| State | Visual |
|-------|--------|
| Unchecked (default) | Checkbox unchecked, row text at normal opacity |
| Checked | Checkbox checked (primary fill), row text at full opacity |
| All checked | "Select All" changes to "Deselect All" |
| No contacts in batch | Empty state shown (see Copywriting below) |
