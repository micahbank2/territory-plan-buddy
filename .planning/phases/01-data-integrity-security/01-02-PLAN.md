Rollback pattern from RESEARCH.md (functional updater — safe under concurrent mutations):
```typescript
const update = useCallback(async (id: string, u: Partial<Prospect>) => {
  const previousProspect = data.find(p => p.id === id);
  setData(prev => prev.map(p => p.id === id ? { ...p, ...u } : p));  // optimistic
  const { error } = await supabase.from("prospects").update(dbFields).eq("id", id);
  if (error) {
    setData(prev => prev.map(p => p.id === id ? { ...p, ...previousProspect } : p));  // rollback
    toast.error("Failed to save — changes not persisted");
    return;
  }
}, [user, data]);
```

IMPORTANT: Including `data` in the useCallback dependency array is required for the `data.find()` snapshot to be current. This is acceptable for Phase 1 — Phase 2 (TanStack Query) will remove this dependency.
